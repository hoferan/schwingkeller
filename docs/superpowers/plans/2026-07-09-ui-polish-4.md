# UI Polish Round 4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix five UI/UX issues from [issue #8](https://github.com/hoferan/schwingkeller/issues/8): a favicon that doesn't match the red/white brand, a zoom-to-fit button whose border doesn't match Leaflet's native control, an oversized mobile drag handle that clips the header when collapsed, an unusable side menu at tablet width / mobile landscape, and an admin-mode pill that clutters the header.

**Architecture:** Six independent-ish tasks touching six files, plus a final manual-verification task. Tasks 1-3 are small, self-contained style fixes (favicon asset, `MapView.tsx`, `Sidebar.tsx`'s handle/peek constant). Tasks 4-5 build the new tablet/landscape sidebar in two layers: Task 4 delivers a fully working, testable collapsible panel (floating arrow tab, tap-to-toggle, tap-outside-to-close); Task 5 layers a continuous free-drag touch gesture on top, mirroring the mobile sheet's existing drag machinery but on the horizontal axis. Task 6 inverts the Topbar's colors in admin mode instead of overlaying a pill.

**Tech Stack:** React 19 + TypeScript + Vite, inline styles (`CSSProperties` objects, no CSS-in-JS library), `lucide-react` icons, Vitest + React Testing Library, Supabase auth via `useAuth()`/`AuthProvider`.

## Global Constraints

- No new npm dependencies.
- No `any` in TypeScript.
- No changes to `theme.ts` tokens — all new values reuse existing tokens (`theme.color.accent`, `theme.color.accentInk`, `theme.radius.sm`, `theme.radius.pill`, `theme.shadow`).
- Keep i18n keys in sync across DE/FR/IT — this plan adds two new keys (`expandSidebar`, `collapseSidebar`) to all three language blocks in `src/i18n/translations.ts`.
- Diff touches only: `public/favicon.svg`, `src/features/map/MapView.tsx`, `src/features/sidebar/Sidebar.tsx`, `src/features/sidebar/Sidebar.test.tsx`, `src/App.tsx`, `src/components/Topbar.tsx`, `src/components/Topbar.test.tsx`, `src/i18n/translations.ts`.
- Run `npm run lint`, `npm run test`, and `npm run typecheck` after every task; all three must pass before committing.
- **Do not push to `origin`.** Commit locally after each task only — push is out of scope for this plan.

**Deviation from spec, noted up front:** the spec (`docs/superpowers/specs/2026-07-09-ui-polish-4-design.md` §3) called for *dynamically measuring* the sidebar's peek height via `headerRef.current.offsetHeight` in a `useEffect`, mirroring `MapView.tsx`'s Leaflet-control measurement. That pattern is right for `MapView.tsx` because it aligns with a third-party control's real DOM size, which the app doesn't control. But the sidebar header's content is entirely first-party and its longest string (`'Schwingkeller suchen'`, DE) is short enough it will never wrap within the 344px panel — and jsdom (this project's test environment) never computes real layout, so `offsetHeight` is always `0` there, which would silently break every existing gesture-threshold test that depends on `PEEK_HEIGHT`'s value. Task 3 below instead hand-computes a corrected static constant with its derivation documented in a code comment — same visible outcome (header fully visible when collapsed), no runtime measurement, fully testable. The spec's other four sections are implemented as written.

The spec's §4 used `transform: translateX(...)` for the tablet panel's slide animation. This plan uses `left` instead (Task 4/5) — for the same reason the mobile sheet already animates `height` rather than a transform: it keeps the panel's on/off-screen state trivially assertable via `element.style.left` in tests (a plain pixel-length string), with no user-visible difference in behavior.

- [ ] **Step 4: Add the tablet touch handlers and the native touchmove listener**

Insert these three handlers and one effect right after the existing `handleTouchCancel` function (`src/features/sidebar/Sidebar.tsx:162-167`) and before the existing mobile `useEffect` that registers the vertical `touchmove` listener (line 169):

```tsx
  const handleTabletTouchStart = (e: React.TouchEvent<HTMLDivElement>) => {
    touchStartXRef.current = e.touches[0].clientX;
    dragStartXRef.current = sidebarOpen ? 0 : -TABLET_PANEL_WIDTH;
    startedOnDragZoneRef.current = !!(
      (headerRef.current && headerRef.current.contains(e.target as Node)) ||
      (tabRef.current && tabRef.current.contains(e.target as Node))
    );
    tabletDraggingRef.current = false;
    dragXRef.current = null;
  };

  const handleTabletTouchEnd = () => {
    const wasDragging = tabletDraggingRef.current;
    touchStartXRef.current = null;
    tabletDraggingRef.current = false;
    startedOnDragZoneRef.current = false;
    // A non-dragging touch on the tab is a real tap — the tab's own onClick handles that. A
    // non-dragging touch on the header (no click target there) is simply ignored, same as mobile
    // ignores a non-header, non-dragging touch.
    if (!wasDragging) return;

    const finalX = dragXRef.current ?? dragStartXRef.current!;
    const travelled = finalX - dragStartXRef.current!;
    setDragX(null);
    dragXRef.current = null;
    if (travelled > TABLET_PANEL_WIDTH * 0.25) {
      onSetSidebarOpen(true);
    } else if (travelled < -TABLET_PANEL_WIDTH * 0.25) {
      onSetSidebarOpen(false);
    } else {
      onSetSidebarOpen(sidebarOpen);
    }
  };

  const handleTabletTouchCancel = () => {
    touchStartXRef.current = null;
    tabletDraggingRef.current = false;
    startedOnDragZoneRef.current = false;
    dragXRef.current = null;
    setDragX(null);
  };

  // Native (non-passive) listener for the same reason the mobile drag effect below uses one:
  // JSX's onTouchMove is passive by default, which silently no-ops preventDefault().
  useEffect(() => {
    if (!isTablet) return;
    const root = rootRef.current;
    if (!root) return;

    const onTouchMove = (e: TouchEvent) => {
      if (touchStartXRef.current === null || !startedOnDragZoneRef.current) return;
      const deltaX = e.touches[0].clientX - touchStartXRef.current;

      if (!tabletDraggingRef.current) {
        if (Math.abs(deltaX) <= 8) return; // sub-slop jitter — still a potential tap
        tabletDraggingRef.current = true;
      }

      e.preventDefault();
      const newX = dragStartXRef.current! + deltaX;
      const clamped = Math.min(0, Math.max(-TABLET_PANEL_WIDTH, newX));
      dragXRef.current = clamped;
      setDragX(clamped);
    };

    root.addEventListener('touchmove', onTouchMove, { passive: false });
    return () => root.removeEventListener('touchmove', onTouchMove);
  }, [isTablet]);
```

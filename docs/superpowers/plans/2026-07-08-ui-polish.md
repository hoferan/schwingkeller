# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Four small, independent visual polish fixes from live-preview feedback: a bigger mobile drag handle, venue list rows restyled as bordered/shadowed cards, and the map/satellite layer picker plus zoom-to-fit button restyled to match Leaflet's own native zoom-control look, with zoom-to-fit repositioned directly beneath the native +/- control.

**Architecture:** Two files change, in four tasks with no cross-task dependencies except within the same file: `Sidebar.tsx` (drag handle, then venue-list-row cards) and `MapView.tsx` (layer-picker native styling, then zoom-to-fit native styling + reposition — the second MapView task reuses a style constant the first one introduces). `src/index.css` gets one small hover-state rule shared by both MapView tasks.

**Tech Stack:** React 19 + TypeScript + Vite, inline styles, Leaflet/react-leaflet, Vitest + React Testing Library.

## Global Constraints

- No new npm dependencies.
- No `any` in TypeScript.
- No changes to `theme.ts`, i18n keys, or component behavior/props — this plan is purely visual/positional (`Sidebar.tsx`, `MapView.tsx`, `src/index.css`).
- `npm run test` remains a pure regression safety net (no test asserts on colors/sizes/layout) — no test files change in this plan.
- Run `npm run lint`, `npm run test`, and `npm run typecheck` after every task; all three must pass before committing.
- **Do not push to `origin`.** Commit locally after each task only — push is out of scope for this plan.

- [ ] **Step 4: Measure the real zoom-control height once the map is ready**

Old (`src/features/map/MapView.tsx`, inside the mount effect):

```tsx
    map.whenReady(() => {
      if (!mapRef.current) return;
      map.invalidateSize();
      window.setTimeout(() => {
        if (!mapRef.current || !markerGroupRef.current) return;
```

New:

```tsx
    map.whenReady(() => {
      if (!mapRef.current) return;
      map.invalidateSize();
      const zoomEl = map.zoomControl.getContainer();
      if (zoomEl) setFitAllTop(10 + zoomEl.offsetHeight + 10);
      window.setTimeout(() => {
        if (!mapRef.current || !markerGroupRef.current) return;
        map.invalidateSize();
        if (!map.hasLayer(markerGroupRef.current)) markerGroupRef.current.addTo(map);
        refreshMarkers();
      }, 160);
    });
```

`map.zoomControl` is typed as `Control.Zoom` on Leaflet's `Map` interface (present because the map is created with `zoomControl: true`); `getContainer()` returns `HTMLElement | undefined`, so the `if (zoomEl)` guard covers the (practically unreachable, but type-required) undefined case. `10 + zoomEl.offsetHeight + 10` reproduces Leaflet's own `10px` top-control margin plus its own `10px` gap between stacked controls, so the fit-all button lines up correctly whether Leaflet renders the `26px` (mouse) or `30px` (touch) zoom control.

# UI Polish Round 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the map/satellite native-radio control with a segmented pill toggle, fix the detail modal's off-center close icon and opaque wappen background, and rebuild the mobile drawer's swipe gesture into a continuous free-drag bottom sheet that tracks the finger and coexists with the scrollable venue list.

**Architecture:** Four independent tasks. Task 1 (`MapView.tsx`) and Task 2 (`DetailModal.tsx`) are pure restyles with no shared state. Task 3 (`Sidebar.tsx` + `Sidebar.test.tsx`) is the large one — it replaces the existing threshold-based swipe classification (from the prior round) with continuous drag tracking, a native (non-passive) `touchmove` listener, and scroll-aware gating so dragging coexists with the venue list's own scrolling. Task 4 is manual verification, including a real touch-emulation smoke test since jsdom can't fully exercise touch gesture physics.

**Tech Stack:** React 19 + TypeScript + Vite, inline styles, `lucide-react`, Vitest + React Testing Library.

## Global Constraints

- No `any` in TypeScript.

- Diff touches only: `src/features/map/MapView.tsx`, `src/features/venue-detail/DetailModal.tsx`, `src/features/sidebar/Sidebar.tsx`, `src/features/sidebar/Sidebar.test.tsx`.
- Run `npm run lint`, `npm run test`, and `npm run typecheck` after every task; all three must pass before committing.

- [ ] **Step 1: Rewrite `Sidebar.test.tsx`**

The file changes as follows relative to its current content:
- **Unchanged, keep exactly as-is:** `'renders canton group names'`, `'filters venues as the user types in the search box'`, `'hides admin tools when not admin'`, `'closes the mobile drawer on tap outside it'`, `'does not close the mobile drawer on tap inside it'`, `'treats a small header touch as a tap toggle'`, `'does nothing in the dead zone between tap and swipe thresholds'`, `'treats exactly 10px as the dead zone, not a tap'`. All eight of these either don't touch touch/swipe behavior at all, or use small deltas (4px, 10px, 18px) that never fire a `touchmove` and never reach the new drag-classification logic — so they exercise exactly the same code path before and after this task and need no changes.
- **Replace** `'opens the drawer on an upward swipe past the threshold'` and `'closes the drawer on a downward swipe past the threshold'` with versions that fire an intermediate `touchMove` (required to enter drag mode under the new model) and use a swipe distance (150px) that's unambiguously past the new 25%-of-range commit threshold regardless of jsdom's default `window.innerHeight` (768, giving a ~506px range and a ~127px threshold — 150px clears it with margin).
- **Replace** `'treats exactly 30px downward as the swipe-close boundary'` and `'treats exactly -30px upward as the swipe-open boundary'` — a fixed 30px is no longer a meaningful threshold in the new percentage-of-range model. Replaced with four tests that explicitly stub `window.innerHeight` to a controlled value so the 25%-of-range math is exact: two that land just past the threshold (commit to the new state) and two new ones that land just short of it (snap back to the starting state) in each direction.
- **Add** three new tests covering the list-body drag-gating behavior (starting a touch on the scrollable list itself, not the header).

```tsx
  const { t } = useTranslation();
  const { isAdmin } = useAuth();
  const fileRef = useRef<HTMLInputElement>(null);
  const rootRef = useRef<HTMLDivElement>(null);
  const headerRef = useRef<HTMLDivElement>(null);
  const listRef = useRef<HTMLDivElement>(null);
  const touchStartYRef = useRef<number | null>(null);
  const dragStartHeightRef = useRef<number | null>(null);
  const openHeightPxRef = useRef(0);
  const isDraggingRef = useRef(false);
  const startedOnHeaderRef = useRef(false);
  const [dragHeight, setDragHeight] = useState<number | null>(null);

  const handleTouchStart = (e: React.TouchEvent<HTMLDivElement>) => {
    touchStartYRef.current = e.touches[0].clientY;
    openHeightPxRef.current = window.innerHeight * 0.8;
    dragStartHeightRef.current = sidebarOpen ? openHeightPxRef.current : PEEK_HEIGHT;
    startedOnHeaderRef.current = !!(headerRef.current && headerRef.current.contains(e.target as Node));
    isDraggingRef.current = false;
  };

  const handleTouchEnd = (e: React.TouchEvent<HTMLDivElement>) => {
    const startY = touchStartYRef.current;
    const wasDragging = isDraggingRef.current;
    touchStartYRef.current = null;
    isDraggingRef.current = false;
    if (startY === null) return;

    if (!wasDragging) {
      // Never became a drag — either a pass-through list scroll, or too small to classify.
      if (startedOnHeaderRef.current) {
        const deltaY = e.changedTouches[0].clientY - startY;
        if (Math.abs(deltaY) < 10) {
          // Suppress the synthetic compat click that follows a touch tap, which would otherwise
          // double-fire onToggleSidebar via the header's onClick.
          e.preventDefault();
          onToggleSidebar();
        }
      }
      return;
    }

    // Snap relative to how far the drag travelled from its OWN starting point, not the sheet's
    // absolute midpoint — the peek-to-open range is large (~500px+), so an absolute-midpoint snap
    // would require a much heavier drag than the old ±30px swipe to ever flip state.
    const finalHeight = dragHeight ?? dragStartHeightRef.current!;
    const range = openHeightPxRef.current - PEEK_HEIGHT;
    const travelled = finalHeight - dragStartHeightRef.current!;
    setDragHeight(null);
    if (travelled > range * 0.25) {
      onSetSidebarOpen(true);
    } else if (travelled < -range * 0.25) {
      onSetSidebarOpen(false);
    } else {
      onSetSidebarOpen(sidebarOpen);
    }
  };

  // Continuous drag tracking. Attached natively (not as a JSX onTouchMove prop) because React
  // registers JSX touchmove listeners as passive by default, which silently makes
  // preventDefault() inside them a no-op — blocking native scroll while dragging requires the
  // { passive: false } form, which JSX doesn't expose.
  useEffect(() => {
    if (!isMobile) return;
    const root = rootRef.current;
    if (!root) return;

    const onTouchMove = (e: TouchEvent) => {
      if (touchStartYRef.current === null) return;
      const deltaY = e.touches[0].clientY - touchStartYRef.current;

      if (!isDraggingRef.current) {
        if (startedOnHeaderRef.current || !sidebarOpen) {
          isDraggingRef.current = true;
        } else {
          const list = listRef.current;
          const atTop = !list || list.scrollTop <= 0;
          if (atTop && deltaY > 5) {
            isDraggingRef.current = true;
          } else {
            return;
          }
        }
      }

      e.preventDefault();
      const newHeight = dragStartHeightRef.current! - deltaY;
      const clamped = Math.min(openHeightPxRef.current, Math.max(PEEK_HEIGHT, newHeight));
      setDragHeight(clamped);
    };

    root.addEventListener('touchmove', onTouchMove, { passive: false });
    return () => root.removeEventListener('touchmove', onTouchMove);
  }, [isMobile, sidebarOpen]);

  useEffect(() => {
    if (!isMobile || !sidebarOpen) return;
    const onPointerDown = (e: PointerEvent) => {
      if (rootRef.current && !rootRef.current.contains(e.target as Node)) {
        onSetSidebarOpen(false);
      }
    };
    document.addEventListener('pointerdown', onPointerDown);
    return () => document.removeEventListener('pointerdown', onPointerDown);
  }, [isMobile, sidebarOpen, onSetSidebarOpen]);
```
